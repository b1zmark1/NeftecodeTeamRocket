## Baseline

Все файлы внутреннего baseline собраны в папке `baseline/`.

Запуск:

```bash
python baseline/train_boosting_baseline.py
```

Скрипт:

- читает данные из `docs/`
- собирает сценарные признаки из состава рецептур и свойств компонентов
- использует `typical` как fallback для свойств партий
- обучает baseline на `CatBoost`
- сохраняет предсказания, метрики и SHAP-артефакты прямо в `baseline/`

Содержимое `baseline/`:

- `train_boosting_baseline.py` — код обучения и генерации артефактов baseline
- `metrics.json` — метрики кросс-валидации на train для двух целевых переменных
- `predictions.csv` — предсказания для тестового набора по `scenario_id`
- `train_features.csv` — итоговая матрица признаков train на уровне сценариев
- `test_features.csv` — итоговая матрица признаков test на уровне сценариев
- `property_alias_map.csv` — соответствие внутренних алиасов `prop_*` реальным названиям свойств
- `*.cbm` — обученные модели CatBoost по каждой целевой переменной
- `*_shap_importance.csv` — таблицы важности признаков по mean absolute SHAP с человекочитаемыми названиями
- `*_shap_bar.png` — глобальная важность признаков для каждой цели
- `*_shap_beeswarm.png` — распределение влияния признаков на предсказания для каждой цели

Замечание: по условиям хакатона деревья и бустинги нельзя использовать как финальный предиктор. Этот код нужен как служебный baseline и инструмент анализа факторов.
